План развития PersonaFeed после хакатона (сдача 11.07.2026, реализация — после).
Синтез трёх источников: комментарии по позиционированию (tg_chat.txt), анализ

## 1. Позиционирование

**Research mode = «разведка контента по нише».** Маркетологу, контентмейкеру,
YouTube-блогеру нужно видеть нишу не через свой персонализированный YouTube
(который показывает то, что он уже смотрел), а сырьём: что вообще выходит по
теме, что набирает просмотры у маленьких каналов, какие форматы всплывают.
Это замена ручного мониторинга YouTube, который они и так делают руками, —
понятный ROI и платящее ядро продукта.

**Честная формулировка.** YouTube Data API не даёт доступа к рекомендательному
алгоритму — «настоящую ленту» чужого человека получить нельзя ни при каких
условиях. Продукт даёт несглаженную выдачу по нише с реальными метриками
(просмотры, каналы, даты) — и это позиционируется явно, без обещания
«реальной ленты персоны».

**Персона — не ядро, а AI-помощник ввода.** Ядро режима — custom-запросы,
которые пользователь задаёт сам. Опционально: поле «опиши свою ЦА»
(«мама 35 лет из Алматы») → Claude превращает описание в набор запросов,
которые пользователь **видит и редактирует** перед запуском.

Ответ на риск «ИИ генерит запросы с нулевой историей ввода» (проверено на
Google Trends): в SEO запрос — это результат, и там ИИ проваливается; здесь
запрос — сеть для вылова. YouTube search работает семантически и возвращает
популярные видео по теме даже при неестественной формулировке. Верификация
происходит на выходе: если по запросу находятся только видео с 200 просмотрами —
запрос отбрасывается автоматически (см. фильтр жизнеспособности ниже).

**Тиры (из tg_chat):**

| Tier | Цена | Что входит | Аудитория |
|---|---|---|---|
| Free | 0 | 19 категорий + 🎲 + «тема дня» | воронка и привычка |
| Pro | $4.99 | custom-ниши, research-фильтры, weekly digest | маркетологи, контентмейкеры |
| Edu/Family | $19.99 | шаринг наборов, управляемые ленты | учителя, родители (позже) |

## 2. Ядро: custom-ниша = динамическая категория

Движок «запрос → пул → случайная сетка» уже есть (`pool.py`, `youtube.py`).
Custom-ниша — это `Category`, созданная пользователем, а не из хардкода
`categories.py`. Продукт не размывается: все юзкейсы сидят на одном движке,
меняется только то, кто задаёт запросы.

### Этап 1 — Custom-ниши (первый доллар)

- **`POST /research/grid`** `{queries: [...], source, limit}` — сетка по нише.
  Пул с ключом `niche:{hash(queries)}:{source}`: кеш шарится между одинаковыми
  нишами, переиспользует `pool.get_pool()` с динамическим cat-dict.
- **`POST /research/suggest`** `{audience: "мама 35 лет из Алматы"}` →
  новый промпт в `ai.py` (по образцу `_TOPIC_PROMPT`) →
  `{queries, queries_ru, region, language, rationale}`. Запросы редактируемы.
- **Заземление реальными данными:** перед генерацией Claude получает тренды
  региона (`videos.list chart=mostPopular&regionCode=XX` — 1 юнит квоты,
  чистые реальные данные) — интересы строятся с оглядкой на то, что реально
  смотрит регион, а не из головы модели.
- **Фильтр жизнеспособности запросов:** после `_enrich()` отбрасывать запросы,
  у которых суммарные просмотры топ-результатов ниже порога. Слабые
  ИИ-запросы отсеиваются метриками автоматически.
- **Валидация ввода:** длина текста ЦА, лимит числа запросов, JSON-only промпт
  (`_parse_json` уже безопасно падает на мусоре), rate limit на suggest.

### Этап 2 — Research-фильтры ✅ реализован в текущей версии (хакатон, 10.07.2026)

- Сортировка по дате / по просмотрам.
- Фильтры: «до 10k просмотров» (поиск восходящих трендов), период публикации —
  **client-side** по уже загруженному пулу (`duration`, `views`, `published_at`
  уже есть в `VideoItem`).
- «Только каналы младше года» — требует `channels.list`
  (`snippet.publishedAt`), бэкенд-обогащение (1 юнит на 50 каналов).
- `youtube.py`: параметризовать `relevanceLanguage` (сейчас захардкожен `"en"`),
  добавить `regionCode` и `order`.

### Этап 3 — Сохранённые ниши + weekly digest

- Сохранённые ниши: localStorage → позже аккаунты.
- Weekly digest на почту: «за неделю в твоей нише вышло 47 видео, топ-5 по
  динамике» — требует снапшотов пула, дельты просмотров и email-инфраструктуры
  (общей с «темой дня» из roadmap). Самый тяжёлый этап, делается последним.

## 4. Квоты и риски

- **Квота YouTube API — главный риск.** `search.list` = 100 юнитов, дефолт
  10 000/день. Одна ниша (5–8 запросов) = 500–800 юнитов → ~12–15 свежих ниш
  в день поверх обновления базовых пулов. Митигации: кеш по хешу запросов
  (одинаковые ниши шарятся между пользователями), TTL 24 ч, лимит запросов
  на нишу, запрос увеличения квоты у Google.
- **Галлюцинации интересов ИИ** (стереотипная ЦА вместо реальной) →
  редактируемость запросов + метрический фильтр + заземление трендами региона.
- **География** — только уровень страны (`regionCode: KZ`), город кодируется
  через сами запросы («алматы куда сходить с детьми»).

## 6. Roadmap остальных предложений (из tg_chat, без детализации)

1. **«Тема дня»** — раз в день лента подмешивает одну категорию, которую
   пользователь НЕ выбирал («сегодня: 🏛 История — 6 видео вне твоего пузыря»).
   Дёшево (бейдж в UI), возвращает миссию «выход из пузыря», растит retention.
   По tg_chat — делать первым.
2. **Daily digest** (email/push): «твоя утренняя сетка готова: 12 свежих видео +
   3 сюрприза». Инфраструктура общая с research-digest.
3. **Education mode** — учитель собирает набор категорий → делится ссылкой с
   классом; родительский режим; «сравни две категории» на одном экране.
   Делать последним, по входящему спросу от учителей.

**Порядок внедрения:** тема дня + digest (retention всем) → custom-ниши
(первый доллар) → education.
